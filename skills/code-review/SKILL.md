# code-review：代码检查

## 触发条件

- 用户要求检查代码质量、审查 PR、review 代码
- 用户要求检查安全性、性能、可维护性

## 执行流程

```text
1. 明确检查范围和关注点
2. 逐文件检查：
   - 逻辑是否正确
   - 是否存在安全隐患（XSS、注入、泄露等）
   - 是否有性能问题（不必要的重渲染、内存泄漏等）
   - 是否遵循项目代码规范
   - 是否有未处理的边界情况
   - 错误处理是否完善
3. 输出问题清单（按严重程度排序）
4. 对每个问题给出具体修改建议
```

## 输出格式

```text
🔴 严重：必须修复
🟡 一般：建议修复
🟢 建议：可选优化

每一项包含：文件路径 + 行号 + 问题描述 + 修改建议
```

## 禁止事项

- 不修改代码（仅检查和建议）
- 不对主观风格选择过度批评
