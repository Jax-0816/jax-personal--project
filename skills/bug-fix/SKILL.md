# bug-fix：Bug 修复

## 触发条件

- 用户报告 bug、反馈异常行为、要求修复问题
- 测试或构建失败

## 执行流程

```text
1. 复述 bug 现象，确认与用户理解一致
2. 定位问题代码，说明根因
3. 提出修复方案（仅修改问题相关代码）
4. 确认修复不会引入新问题或影响已有功能
5. 等待用户确认
6. 修改代码
7. 验证修复后建议测试命令
```

## 输出要求

- 问题根因（为什么会出现这个 bug）
- 修复了哪个文件、哪几行
- 为什么这样修复
- 建议的验证步骤

## 禁止事项

- 不借机重构无关代码
- 不扩大修改范围
- 不引入新的依赖
