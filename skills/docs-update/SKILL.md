# docs-update：文档更新

## 触发条件

- 用户要求更新 README、TODO、AGENT 等文档
- 功能变更后需要同步更新文档
- 新增文件需要记录到项目文档

## 执行流程

```text
1. 确认需要更新的文档和更新原因
2. 读取现有文档内容
3. 说明准备修改的部分和修改内容
4. 等待用户确认
5. 修改文档（只改与当前任务相关的内容）
6. 输出变更对比
```

## 更新范围

- README.md：项目说明、使用方法、项目结构
- TODO.md：任务进度、待办事项、下一步计划
- AGENT.md：开发规则、Skill 定义
- .env.example：环境变量模板
- docs/：设计文档、开发日志

## 输出要求

- 更新了哪些文档
- 每个文档具体改了什么
- 是否需要同步更新其他相关文档

## 禁止事项

- 不修改与当前任务无关的文档内容
- 不删除已有文档记录
- 不擅自创建用户未要求的文档
