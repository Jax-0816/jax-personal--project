# backend-dev：后端开发

## 触发条件

- 用户要求开发后端 API、数据库操作、服务端逻辑
- 任务涉及 Node.js、Python、数据库、接口设计

## 执行流程

```text
1. 理解接口需求和数据结构
2. 确认技术选型（框架、数据库、认证方式）
3. 设计接口：请求/响应格式、错误码、边界条件
4. 列出计划修改和新建的文件
5. 说明对现有接口的影响
6. 等待用户确认
7. 先定义数据结构，再写接口逻辑，最后加错误处理
8. 完成后建议接口测试方式
```

## 输出要求

- 接口定义（Method、Path、Request/Response）
- 数据结构说明
- 错误处理策略
- 建议的测试命令或 curl 示例

## 禁止事项

- 不暴露敏感信息（密钥、密码、Token）
- 不跳过输入校验
- 不使用不安全的 SQL 拼接
- 不引入不必要的新依赖
